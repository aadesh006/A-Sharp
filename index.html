A-Sharp (A#)

A high-performance, stack-based bytecode virtual machine written in strictly compliant C99.

Architecture & Internal Subsystems

Bytecode Execution Engine

A-Sharp operates on a custom instruction set architecture. The single-pass compiler translates source text directly into dense, cache-friendly bytecode chunks, entirely bypassing abstract syntax tree (AST) generation. The VM executes this bytecode via a highly optimized, flat array stack, ensuring minimal pointer indirection and high instruction throughput.

Mark & Sweep Garbage Collector

Memory safety is enforced via a dynamic, threshold-triggered garbage collector. The GC traverses the live object graph starting from root pointers (globals, stack frames, open upvalues), marking reachable allocations. It employs a tricolor abstraction (white, gray, black) to systematically sweep unreachable heap allocations, eliminating memory leaks during intense workloads.

Lexical Scoping & Closures

The language supports first-class functions and deep lexical scoping. Local variables are resolved entirely at compile time and accessed via exact stack array indices (`OP_GET_LOCAL`). Variables captured by closures are seamlessly hoisted to the heap using an `ObjUpvalue` linked list, maintaining state even after the parent execution context is destroyed.

Dynamic String Interning

To reduce heap fragmentation and accelerate execution, all strings in A-Sharp are automatically interned into a global hash table. This guarantees string uniqueness in memory, allowing the VM to evaluate string equality using O(1) pointer comparisons rather than O(n) character-by-character checks.

Syntax Example: GC Stress Test

This script demonstrates dynamic heap allocation and automatic garbage collection. The loop rapidly allocates 100,000 concatenated strings. Because `dump` goes out of scope per iteration, the GC intercepts the allocations dynamically, sweeping the unreferenced pointers before the heap threshold is exhausted.

var a = "garbage";
var b = " collection";
var i = 0;

while (i < 100000) {
  // Dynamically allocates a new string on the heap every loop iteration.
  var dump = a + b; 
  i = i + 1;
}

print "Execution complete. GC prevented OOM crash.";

System Installation

Debian / Ubuntu (APT Package Manager)

A-Sharp is distributed as a pre-compiled `amd64` `.deb` package. Execute the following commands to add the repository to your system and install the binary globally.

# 1. Authorize the A-Sharp APT repository
echo "deb [trusted=yes] https://aadesh006.github.io/A-Sharp/ ./" | sudo tee /etc/apt/sources.list.d/asharp.list

# 2. Synchronize package indices
sudo apt update

# 3. Install the executable
sudo apt install asharp

macOS & Unsupported Linux Architectures (Build from Source)

For systems operating on `arm64` (Apple Silicon) or distros outside the Debian family, the C toolchain is required to build the VM locally.

# Clone the repository and compile
git clone https://github.com/aadesh006/A-Sharp.git
cd A-Sharp
make

# Execute the compiled binary
./asharp
